CI/CD Pipeline for Containerized App on AWS

Terraform • GitHub Actions • Docker • AWS EC2 • ECR • OIDC

📌 Overview

This project demonstrates a fully automated CI/CD pipeline that deploys a containerized application to AWS EC2 using Terraform and GitHub Actions. It follows real-world DevOps practices including Infrastructure as Code, secure authentication, and automated deployments.

⚙️ How It Works

1. Infrastructure (Terraform)

  • VPC, Subnets, Security Groups
  • EC2 provisioning
  • ECR repository setup
  • IAM roles & policies
  • S3 remote state + locking

2. CI/CD Pipeline

  • Triggered on GitHub push
  • GitHub Actions workflow
  • OIDC authentication (no secrets)
  • Docker image build
  • Push to Amazon ECR

3. Deployment

  • EC2 pulls latest image from ECR
  • Runs container via Docker
  • App exposed via public IP
  • Automated updates on push

🧰 Tech Stack

Terraform AWS EC2 VPC IAM ECR S3 Docker GitHub Actions Linux VS Code

🔐 Security Highlights

  • No AWS credentials stored in GitHub
  • OIDC-based authentication
  • Least privilege IAM roles
  • Remote Terraform state in S3 with locking

🚀 Terraform Workflow

Step 1: Bootstrap Backend

cd backend
terraform init
terraform apply

Step 2: Deploy Infrastructure

cd terraform
terraform init
terraform apply
